Introduction

MyFacebook is a Java program meant to simulate a social media website. The goal of this program is to give a basic understanding of how permissions work with respect to access control found in operating systems. Users can create their profile, add friends, add lists, post pictures, and comment/view those pictures. However, owners of the profile and pictures can modify the permissions of those who can read/modify them. The program is executed from the terminal using the following form:

java access filename.txt

Where filename.txt is a text file from which the commands are executed.

Start Up

In order to start up the program:

  • A profile owner must be created.
  • This owner must be the active user.

Otherwise, the program will terminate.

To accomplish this, the friendadd and viewby commands are used:

friendadd friendname
viewby friendname

The program must begin with these two commands. Friendadd will add the given friendname to the friends.txt file. This file keeps a list of friends. When executed in the beginning of the program, the first added friend becomes the profile owner. The viewby command then sets the active user to the given friendname. In the beginning, this must be the profile owner.

Adding Friends

Once you have correctly created and activated the user, you may begin using additional commands. You may now use the friendadd command to add new friendnames to the friends.txt file. These friends will then be able to become active users and can preform the following actions given they have the correct permissions:

viewby friendname
postpicture picturename.txt
writecomments picturename.txt text
readcomments picturename.txt
Adding/Modifying Lists

The profile owner, however, has additional commands. He or she can also create lists to which friends and photo can then be added. This is included so that the members of these lists can be given certain permissions

listadd listname
friendlist friendname listname

Listadd is used to create a list with the given listname, while friendlist adds the friendname to the listname, given that both are valid parameters. These command can only be executed if:

  • The current user is the profile owner.
  • The friendname is a friend in the friends.txt file.
  • The listname is a list in the in-memory list map.
Posting Pictures

To post a picture is to create a text file to represent the picture. The text file will then contain the name of the picture, as well as the comments on the picture. The picture will be created with no list associated, the owner will be the current user, and the permissions will be set to rw -- --. In order to post a picture, however:

  • The current user must have permissions to post a picture.
  • A picture with the same name must not exist.

The user can then change the permissions of a picture using:

chlst picturename.txt listname
chmod picturename.txt -- -- --
chown picturename.txt ownername

Only the profile owner or the picture owner can issue these commands. Chlst modifies the list associated with the picture, chmod modifies the permissions, and chown changes the owner of the picture.
